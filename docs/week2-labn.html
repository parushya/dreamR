3. Objects and Datatypes

Everything in R is called an “object”

Objects contain data. R has five basic classes/datatypes:

  1. Character - (“abc”)
  2. Numeric - (real numbers: 1, 7.5)
  3. Integer - (whole numbers: 1, 2, -896)
  4. Logical - (TRUE, FALSE)
  5. Complex - (1+0i, 2+4i)

4. Data Structures

Vectors

Vectors store elements of the same datatype.

We can create a vector using the c() command.

a_num <- c(0,0.7,9,2,3,4,-1)            # numeric or double

b_logical <- c(TRUE,FALSE,TRUE,TRUE,TRUE) # logical

c_logical <- c(T,F,T,T,T) # also logical - Never use T and F as it leads to errors in analysis

d_char <- c("Sheila", "Nila", "Camilla")  # character

e_int <- 1:20 # integer

f_int <- c(1,2,3,4,5)  # integer

g_int <- c(1+0i,2+4i) # complex numbers

Accessing Elements in Vectors

a_num[1]  # First element
a_num[3]  # Third element
a_num[1:3] # First three elements

Lists

Accessing elements in a List

If we want to create something that stores objects of different classes together, we use another data structure called list.

A list can contain two or more classes of objects with different lengths.

Creating lists

list_1 <- list("a" = 2.5, "b" = TRUE, "c" = 1:3)

list_1
$a
[1] 2.5

$b
[1] TRUE

$c
[1] 1 2 3

We created a list with objects of three different types - numeric, logical, and integer vector.

# Structure of the list
str(list_1)
List of 3
 $ a: num 2.5
 $ b: logi TRUE
 $ c: int [1:3] 1 2 3

We can also create a list with existing vectors.

# A new vector
name_vec <- c("Camilla","Nila","Duflo","Akbar")

# And then lets use the vectors we already have in the environment
list_2 <- list(name_vec, c_logical, d_char, f_int, e_int, a_num)
list_2
# let's check the classes of objects
class(list_2[[2]])
class(list_2[[3]])

# And their lengths
length(list_2[[2]])
length(list_2[[3]])

Factors

Factors are used for categorical data - both nominal and ordinal ones.

Factors are treated as a separate datatype in R. Technically, factors are stored as a vector of integer values corresponding to the character type objects which they are used to represent.

You can define a factor by using factor() command.

vec_1 <- c("yes", "no", "yes")

fct_1 <- factor(c("yes", "no", "yes"))

# Notice the difference in outputs

vec_1

fct_1
# Btw, table() command cn be used in R for cross-tabulations
# with both vector and factor datatypes. 

table(vec_1)
table(fct_1)

Ordering Factors

Sometimes it is essential to specify the orders of your factor levels. Particularly during modelling and estimation with binary or categorical variables, given that the first level of factor is used in most functions, like lm(linear regression command in R), will be treated as baseline level or category.

For example, we have a variable measuring dose of vaccine administered (placebo, medium, high). Here specifying order becomes important as all measurements of the treatment efficacy will have to be with respect to the baseline category.

We use levels() within factor() command to do this.

fct_2 <- factor(c("High", "High", "Medium", "Medium", "High", "High","Placebo"))
fct_2   

# (Order is often determined using alphabetical variables by default) (H-M-P)

# Ordering it
fct_2 <- factor(c("Placebo", "High", "Medium", "Placebo", "Medium", 
            "Medium", "High", "High"),
levels = c("Placebo","Medium","High"))

fct_2

Data frames store data in a tabular format.

df_1 <- data.frame(
    Foo = 15:18,

Illustrative Example: Drug X and Ailment Y Study

Example Code

Copy this code in a new R Script and run line-by-line

# Illustrative Example: Drug X and Ailment Y Study

# Setting up the population
set.seed(123)  # For reproducibility
population_size <- 10000

# Creating a data frame representing the population
drug_study_population <- data.frame(
    id = 1:population_size,  # Integer type: unique identifier for each individual
    treatment_group = sample(c("Drug X", "Placebo"), population_size, replace = TRUE),  # Categorical variable
    pain_level = sample(c("Mild", "Moderate", "Severe"), population_size, replace = TRUE, prob = c(0.5, 0.3, 0.2)),  # Ordered categorical variable
    temperature = round(rnorm(population_size, mean = 37, sd = 1), 1),  # Numeric (continuous) variable: body temperature
    dosage = sample(c(50, 100, 150), population_size, replace = TRUE),  # Numeric (discrete) variable: dosage in mg
    recovery_time = round(rnorm(population_size, mean = 7, sd = 2), 1)  # Numeric (continuous) variable: days to recover
)

# Viewing the first few rows
head(drug_study_population)

# Taking a random sample of 500 individuals from the population
sample_size <- 500
sample_data <- drug_study_population[sample(1:population_size, sample_size), ]

# Checking summary statistics of the sample
summary(sample_data)

# Analyzing the mean recovery time for each treatment group
aggregate(recovery_time ~ treatment_group, data = sample_data, mean)

# Converting pain_level into an ordered factor (Ordinal categorical variable)
sample_data$pain_level <- factor(sample_data$pain_level, levels = c("Mild", "Moderate", "Severe"), ordered = TRUE)

# Checking the structure of the modified dataset
str(sample_data)  # Displays data types of all variables

# Saving the sample data to a CSV file
write.csv(sample_data, "drug_study_sample.csv", row.names = FALSE)

# Reading the saved file back into R
drug_sample_read <- read.csv("drug_study_sample.csv")

# Displaying first few rows of the imported dataset
head(drug_sample_read)
